1""" 

2taguchi: Taguchi experiments are quick and dirt designs that are effective for quality control in 

3engineering. Some experimenters use Taguchi Method designs as screening experiments in a similar 

4manner to Plackett Burman designs. They are useful designs in quality control but due to recent fads 

5in marketing are often misapplied in other fields. Taguchi methods were some of the first 

6`Robust Parameter Designs`. 

7 

8The Taguchi Method is a multi-step process for producing a design. To fully understand the process 

9see the reources mentioned below. We provide a quick overview here: First split your factors into 

10A) inner array factors (factors you control) and B) outer array factors (environmental factors out 

11of your control). The function `recommend_inner_array` will help identify an orthogonal array 

12to use as your design base. Once you know which array you want to use as your base, the function 

13`fetch_inner_array` will retrieve that array. If there are non-trivial interactions you need to 

14model in your inner array, you may need to censor certain columns before proceeding. Finally, 

15append the outer array to your inner array using the function `append_outer_array`. 

16 

17An example workflow is shown below. 

18 

19Unlike traditional factorial designs like those in the module `lind.design/factorial`, Taguchi 

20designs rely heavily on pre-experiment judgement. This is often useful in quality control, but can 

21introduce dangerous assumptions in other fields. Taguchi's method builds on the theory of fractional 

22factorial designs, depending on the experiment requirements, either methodology may be more 

23efficient and/or less biased. It is up to the experimenter to understand when to use each method. 

24 

25Workflow Example 

26----------------- 

27>>> # find the appropriate orthogonal array to build inner array of experiment design 

28>>> print( 

29>>> recommend_inner_array( 

30>>> levels_group_a=2, 

31>>> num_factors_group_a=2, 

32>>> interactions_group_a=1, 

33>>> measured_moments=1 

34>>> )) # shows applicable designs 

35>>> # fetch pandas data frame for inner array 

36>>> inner_array = fetch_inner_array(design_file_name="L4_2_3") 

37>>> # print taguchi line graph of inner array interactions 

38>>> interaction_diagrams("L4_2_3") 

39>>> # censor columns for interactions assumed to be significant 

40>>> inner_array = inner_array.drop(columns=["C (A:B)"]) 

41>>> # append noise factors (outer array) to produce full experiment design 

42>>> experiment_design = append_outer_array(inner_array, 

43>>> noise_factors=[[-1, 1], [-1, 0, 1], [3.4, 6.7, 1.9]]) 

44 

45References 

46---------- 

47Taguchi, Chowdhury, and Wu 

48 * Taguchi's Quality Engineering Handbook (Appendix C) 

49Experiment Design and Analysis Reference (reliawiki) 

50 * Experiment Design and Analysis Reference (Appendix D) 

51

76def recommend_inner_array(levels_group_a: int = 0, num_factors_group_a: int = 0, 

77 levels_group_b: int = 0, num_factors_group_b: int = 0, 

78 interactions_group_a: int = 0, 

79 interactions_group_b: int = 0, 

80 measured_moments: int = 1) -> DataFrame: 

81 """ 

82 recommend_inner_array 

83 

84 It is sometimes difficult to know which orthogonal array to use when defining the inner array 

85 of a Taguchi experiment. This function can help narrow down the appropriate choices available 

86 to an experimenter. The experimenter will often want to choose the design with the fewest 

87 number of runs, but that may not always be the case. 

88 

89 If all arguments are left at default values then the function will return all supported 

90 Taguchi designs. See module docstring for references and examples. 

91 

92 Parameters 

93 ---------- 

94 levels_group_a: int, optional 

95 

96 num_factors_group_a: int, optional 

97 levels_group_b: int, optional 

98 num_factors_group_b: int, optional 

99 interactions_group_a: int, optional 

100 interactions_group_b: int, optional 

101 measured_moments: int, optional 

102 

103 Returns 

104 ------- 

105 pd.DataFrame 

106 A data frame describing the available designs that meet the user's criteria; if the data 

107 frame is empty then no Taguchi orthogonal arrays meet the user's specification 

108 """ 

109 

110 if _sfap is None: 

111 raise Exception("Missing dependency lind-static-resources") 

112 

113 toc = read_csv(_sfap + "/taguchi/taguchi_toc.csv", header=0) 

114 toc = toc[["design_array_name", "design_file_name", "num_runs", "2", "3", "4", "5", "6"]] 

115 toc = toc.dropna() 

116 

117 if num_factors_group_a == 0: 

118 return toc 

119 

120 assert levels_group_a >= 2 and num_factors_group_a >= 2 

121 assert levels_group_b >= 2 or num_factors_group_b == 0 

122 

123 # dev note: should we log the degrees of freedom? 

124 # total number of experiments is lower bounded by dof of experiment 

125 # dof_a = (levels_group_a-1) * num_factors_group_a + interactions_group_a + (measured_moments-1) 

126 # dof_b = (levels_group_b-1) * num_factors_group_b + interactions_group_b 

127 

128 toc = toc[toc[str(levels_group_a)] >= num_factors_group_a + interactions_group_a + 

129 (measured_moments - 1)] 

130 if num_factors_group_b != 0: 

131 toc = toc[toc[str(levels_group_b)] >= num_factors_group_b + interactions_group_b] 

132 

133 return toc

136def fetch_inner_array(design_file_name: Optional[str] = None) -> DataFrame: 

137 """ 

138 fetch_inner_array 

139 

140 The inner array of a Taguchi design is often just a fractional factorial design very similar 

141 to more traditional Fisher factorial designs. These are represented by orthogonal arrays that 

142 are then censored used the experimenter's prior judgement about factor interactions. 

143 

144 See module docstring for references and examples. 

145 

146 Parameters 

147 ---------- 

148 design_file_name: str, optional 

149 the name of the design / design file name that the experimenter wants to use for the inner 

150 array of their design 

151 

152 Returns 

153 ------- 

154 pd.DataFrame 

155 a data frame representing the orthogonal array of interest; users may need to censor 

156 columns in this array to represent prior knowledge about number of factors or number of 

157 significant interactions 

158 """ 

159 

160 if _sfap is None: 

161 raise Exception("Missing dependency lind-static-resources") 

162 

163 design_file_name = design_file_name + ".csv" if design_file_name is not None \ 

164 else "taguchi_toc.csv" 

165 return read_csv(_sfap + "/taguchi/" + design_file_name, header=0)

168def append_outer_array(inner_array: DataFrame, noise_factors: List[List], 

169 noise_factor_names: Optional[List[str]] = None) -> DataFrame: 

170 """ 

171 append_outer_array 

172 

173 The outer array of a Taguchi design represents the noise factors that the designer wants to 

174 control for with respect to the inner array. The outer array represents mini-full factorial 

175 designs each centered at the individual runs in the inner array. This helps provide data that 

176 is "robust" to the levels of the noise factors. 

177 

178 See module docstring for references and examples. 

179 

180 Parameters 

181 ---------- 

182 inner_array: pd.DataFrame 

183 a dataframe representing the inner array of the Taguchi design 

184 noise_factors: List[List] 

185 a list of lists representing the noise factors and levels 

186 noise_factor_names: List[str], optional 

187 a list of names for the factors in the first argument. Must share the order of the first 

188 argument. 

189 

190 Returns 

191 ------- 

192 pd.DataFrame 

193 a combination of the inner and outer arrays representing the full taguchi design; uses the 

194 same format as all factorial designs in this package as opposed to more traditional 

195 Taguchi design display formats 

196 """ 

197 

198 outer_array = _dff( 

199 factors=noise_factors, 

200 factor_names=noise_factor_names 

201 ) 

202 

203 return inner_array.assign(key=0).merge(outer_array.assign(key=0), how='left', on='key').drop( 

204 columns=["key"])
